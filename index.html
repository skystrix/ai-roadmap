Transformers & Large Language Models
The core of applied AI. Master this and you're in the top 10%.
⏱ Month 4–5 · ~240 hours
This phase is the heart of the roadmap. LLMs are the defining technology of the decade. Master transformers, understand the key model families, and know how to use them effectively in production. This is where you go from "knows ML" to "applied AI engineer."

Transformer Architecture (Deep Dive)

Every Component — No Shortcuts
  • Self-Attention — Q, K, V matrices, attention scores, softmax, scale by √d_k. Implement from scratch.
  • Multi-Head Attention — why multiple heads? Each head attends to different patterns.
  • Positional Encoding — sinusoidal (original), learned, RoPE (Rotary), ALiBi
  • Feed-Forward layers — two linear layers with GELU/SiLU, why they matter
  • Layer Normalization — pre-norm vs post-norm, why pre-norm is now standard
  • Residual connections — gradient highway, enables deep networks
  • Encoder vs Decoder vs Encoder-Decoder — BERT vs GPT vs T5/BART
  • KV Cache — how inference is accelerated, memory implications
  • Flash Attention — memory-efficient attention, why it matters for long contexts
  • Grouped Query Attention (GQA) — what LLaMA-2/3 uses, efficiency gains
  • Mixture of Experts (MoE) — Mixtral, GPT-4 speculation, sparse activation

⏱ Month 5–7 · ~240 hours
Market insight: RAG and fine-tuning are the bread and butter of applied AI engineering. Nearly every company building AI products needs engineers who can design, implement, and optimize retrieval systems and customize LLMs. Master these and you have a direct path to $200k+ roles.

Retrieval-Augmented Generation (RAG)

RAG System Components — Master Each
  • Document ingestion — PDF parsing (PyMuPDF, unstructured.io), HTML extraction, table handling
  • Chunking strategies — fixed-size, sentence, semantic, hierarchical. Know tradeoffs.
  • Embedding models — text-embedding-3-small (OpenAI), all-MiniLM, BGE, E5, Cohere embed
  • Vector stores — Chroma (local), Pinecone (cloud), Weaviate, Qdrant, pgvector
  • Retrieval strategies — similarity search, MMR (diverse retrieval), hybrid (BM25 + semantic)
  • Reranking — Cohere Rerank, cross-encoders, ColBERT
  • Context assembly — prompt construction, context window management
  • Advanced RAG — HyDE, query rewriting, multi-query retrieval, RAPTOR
  • RAG evaluation — RAGAS framework, faithfulness, context relevance, answer relevance
  • Graph RAG — Microsoft's GraphRAG, knowledge graph integration
Vector Databases to Know
  • pgvector — Postgres extension. Best for teams already on PostgreSQL. IVFFlat, HNSW indexes.
  • Chroma — easiest to start, great for prototyping, can run locally
  • Pinecone — managed cloud, simple API, used by many startups. Learn upsert/query/filter.
  • Weaviate — hybrid search built-in, GraphQL API, modules system
  • Qdrant — Rust-based, very fast, excellent filtering, recommended for production
  • Milvus — large-scale, enterprise-grade, billion-vector scale
  • FAISS — Facebook's library, in-memory, understand IVF and HNSW indexing algorithms

RAG Orchestration Frameworks

LangChain & LlamaIndex — Study Both
  • LangChain — chains, LCEL (LangChain Expression Language), document loaders, retrievers, memory. Learn how to build without it too (to understand the abstraction).
  • LlamaIndex — best for document-heavy RAG, query engines, response synthesizers, advanced retrieval
  • Haystack (deepset) — production-grade, pipeline-based, good for enterprise
  • DSPy — programmatic LLM optimization, replaces hand-crafted prompts, essential for serious RAG

Fine-Tuning LLMs

Fine-tuning Techniques
  • Full fine-tuning — when to use, compute cost, catastrophic forgetting
  • LoRA (Low-Rank Adaptation) — the most important technique. Understand the math. Rank, alpha hyperparameters.
  • QLoRA — quantized LoRA, fine-tune 65B models on a single 48GB GPU
  • PEFT library — HuggingFace's PEFT, LoraConfig, get_peft_model
  • Instruction tuning — SFT with Alpaca/ShareGPT format data
  • RLHF pipeline — SFT → Reward Model → PPO (trl library)
  • DPO (Direct Preference Optimization) — simpler alternative to RLHF
  • Dataset creation — synthetic data with GPT-4, data curation, deduplication
  • Evaluation after fine-tuning — benchmarks, domain-specific evals

INT
Interview Preparation
Top-tier AI engineering interviews have 4 rounds. Prepare for all.
The 4 interview tracks: (1) ML/AI theory and intuition, (2) Coding and data structures, (3) ML system design, (4) Behavioral + project deep-dive. Most candidates over-prepare coding and under-prepare ML system design. Don't make this mistake.

Round 1: ML/AI Theory

Most Asked Questions at Top Companies
Explain how attention mechanism works. Why do we scale by √d_k?
What is the difference between BERT and GPT? When would you use each?
How does LoRA work mathematically? What do rank and alpha control?
Explain gradient descent vs Adam. What problem does Adam solve?
What is catastrophic forgetting and how do you mitigate it?
How does KV caching work and what are its memory implications?
Explain the difference between supervised fine-tuning and RLHF.
What is hallucination in LLMs and what strategies reduce it?
How does RAG differ from fine-tuning? When do you use each?
Explain cross-entropy loss. Why do we use it for language modeling?
What is perplexity? How do you evaluate a language model?
Describe the transformer's feed-forward layer. What does it learn?

Round 2: Coding (LeetCode + ML Coding)

LeetCode Focus Areas
  • Arrays & Hashing — 20 medium problems minimum
  • Two Pointers, Sliding Window — streaming/text processing patterns
  • Trees & Graphs — BFS/DFS, frequently asked
  • Dynamic Programming — common in ML algorithm questions
  • Target — 150 medium problems, 30 hard. Focus on patterns, not quantity.
  • Python fluency — list comprehensions, generators, defaultdict, heapq
ML Coding Tasks (Must Practice)
  • Implement attention mechanism from scratch (numpy only)
  • Implement k-means clustering from scratch
  • Build a simple tokenizer (BPE or character-level)
  • Implement backpropagation for a 2-layer MLP
  • Write a training loop with mixed precision in PyTorch
  • Build a simple RAG pipeline without using LangChain
  • Implement cosine similarity search over embeddings

Round 3: ML System Design

Practice These Design Problems
Design a RAG system for a company's internal knowledge base with 10M documents. Handle 1k QPS.
Design an LLM serving infrastructure that minimizes cost while meeting P99 latency of 3 seconds.
Design a multi-agent system for automated customer support that escalates to humans when needed.
Design a fine-tuning pipeline for customizing LLMs for different enterprise customers (multi-tenant).
Design a system to detect prompt injection attacks in a public-facing LLM application.
Design an AI-powered code review system for a GitHub-scale code platform.
Design a recommendation system using LLM embeddings at Pinterest/Netflix scale.
ML System Design Framework: Requirements (functional/non-functional) → Data pipeline → Model selection (build vs buy) → Training/fine-tuning approach → Serving infrastructure (latency, scale) → Evaluation metrics → Monitoring & retraining → Cost analysis. Always address tradeoffs explicitly.

Round 4: Behavioral + Project Deep-Dive

Prepare These Stories (STAR Format)
Tell me about a time you had to make a technical decision with incomplete information. (Tradeoffs)
Describe your most technically challenging project. What would you do differently?
Walk me through your RAG system / fine-tuning project. What was the hardest part?
How do you stay current with the rapidly changing AI landscape?
Tell me about a production incident. How did you diagnose and resolve it?
How do you approach building AI systems that need to be reliable and safe?
